# 📋 Project Status: Decentralized Voting dApp
Last Updated: 2026-02-18

## 🚀 Recent Accomplishments (High-Concurrency & Reliability Fixes)
1. **Relayer & Nonce Management**:
   - Implemented **Random Jitter** (0-1000ms) to desynchronize concurrent serverless requests.
   - Added a **5-attempt Retry Loop** for nonce collisions and "underpriced" errors.
   - Boosted **Gas Pricing** by 25% to ensure immediate blockchain inclusion.
2. **Pre-Vote Validity Checks**:
   - Integrated real-time contract queries in `Voter.js` (frontend) to verify registration, allowed status, and "voted" status *before* signature prompting.
   - Eliminates useless signing for voters who have already voted or are not registered for the current version.
3. **Mobile & Network Resilience**:
   - Fixed the "Network Loop" by replacing reloads with a smart polling mechanism in `switchNetwork`.
   - Guaranteed clean sessions via `sdk.terminate()` in the disconnect flow.
4. **Enhanced UI & Feedback**:
   - Implemented **Optimistic UI Updates** for registration actions.
   - Added automated **Fallback Avatars** for candidates with broken/slow IPFS images.
   - Clearer error reporting: Opaque relay errors now translated to human-readable reasons (e.g., "Election has been reset").

## 🐛 Resolved Issues
1. **Simultaneous Voting (Vote Rejection)**: Resolved "Nonce Race Conditions" in Vercel serverless functions that caused parallel votes to fail.
2. **"failed call()" Opaque Errors**: Fixed by catching and decoding revert reasons during relayer simulation.
3. **Double-Voting signature requests**: Prevented by checking the `voters` mapping before triggering MetaMask.
4. **Broken Candidate Images**: Fixed using `ui-avatars.com` fallback logic.
5. **Mobile Deep-Link Session Stuck**: Resolved by forcing session termination on disconnect.
